# Task 3 – SQL Basics (Chinook Database)

## Overview
This task focuses on practicing fundamental SQL concepts such as filtering, sorting, aggregations, grouping, and joins using the Chinook sample database. The goal is to build confidence in writing basic SQL queries commonly used by data analysts.

## Dataset
Chinook Sample Database (SQLite)

The Chinook database represents a digital media store and contains tables such as Customer, Invoice, InvoiceLine, Track, Album, and Artist.

## Tools Used
- SQLite Online (browser-based SQL editor)
- Chinook Example Database

## SQL Concepts Covered
- SELECT queries for data exploration
- Filtering data using WHERE
- Sorting results using ORDER BY
- Aggregations using SUM, AVG, and COUNT
- Grouping data using GROUP BY
- Filtering aggregated results using HAVING
- Pattern matching using LIKE
- Date range filtering using BETWEEN
- Joins between multiple tables
- Ranking customers based on total spend

## Key Queries Implemented
- Customers filtered by country
- Top invoices by total amount
- Total and average sales
- Sales grouped by billing country
- Countries with total sales greater than a threshold
- Top 5 customers by total purchase value

## Files Included
- `queries_task3.sql` – Contains all SQL queries written for this task
- `sales_summary.csv` – Exported summary of total sales by country
- `README.md` – Task explanation and documentation

## Learning Outcomes
- Clear understanding of the difference between WHERE and HAVING
- Practical experience with GROUP BY and aggregate functions
- Improved confidence in writing analytical SQL queries
- Exposure to joining multiple tables to answer business questions

## Conclusion
This task helped reinforce SQL fundamentals required for data analyst roles and provided hands-on experience with real-world style datasets and queries.
